字符串哈希

定义一个字符串到整数的映射 fff，称为 Hash 函数。Hash 的核心思想是将输入的字符串转化为整数方便比较。

KMP

其实 KMP 本身用于字符串匹配其实实用性价值不大，因为字符串哈希也能做到相同的复杂度。KMP 最有优势的地方是它能快速求出 sss 任意前缀的最长 border（既是一个串的真前缀又是一个串的真后缀）。

思想

考虑不难发现，我们假如说已经求出 [1,k][1,k][1,k] 的所有最长 border 长度，而有一个数组记录了这些长度，考虑如何求出第 k+1k+1k+1 位置的最长 border 长度。

假定字符串从 000 开始。（虽然图是从 111 开始的）

一个非常 trival 的想法是，最优情况下，k+1k+1k+1 位置的 border 可以直接继承前面 pikpi_kpik​ 的 border 长度，那么就只需要比较 spiks_{pi_k}spik​​ 与 sk+1s_{k+1}sk+1​ 就行了？

是的。

但如果不匹配呢？

不难发现，图中绿色的部分就是 [0,k][0,k][0,k] 的最长 border。

如果最长 border 不匹配怎么办？那就找次长 border 再来试试！

如何找到次长 border 呢？考虑次长 border 一定是左边绿色部分的真前缀，也是右边绿色部分的真后缀。而绿色部分是一样的！！！

所以我们得到了次长 border 是最长 border 的最长 border，也就是说第 kkk 长 border 是第 k−1k-1k−1 长 border 的最长 border。

那么暴力找到这些 border 然后一个一个看能不能与下一位 sk+1s_{k+1}sk+1​ 匹配即可。

exKMP

可以在线性时间内跑出一个串的每个后缀和本身的最长前缀长度（即 zzz 数组），通常用于优化 dp，但是很少有题会卡哈希 + 二分。

思想

思想和 KMP 本质差不多，就是用现在已有的信息优化求出剩下的信息。

加入我们现在已经求出 z[0⋯i]z[0\cdots i]z[0⋯i] 了，我们来想如何用前面的信息求出 z[i+1]z[i+1]z[i+1]。

我们称上一次匹配的区间为 Z-box，显然，如果目前 i+1>ri+1>ri+1>r，那么这个点与之前的所有的比较都无关，需要暴力跑出该点的 zzz 值。

而 i+1≤ri+1\leq ri+1≤r，情况就不太一样了。

不难发现，此时的字符串是长成这样的。由于我们保证 i+1>li+1>li+1>l，所以有 s[i+1⋯r]=s[i+1−l⋯r−l]s[i+1\cdots r]=s[i+1-l\cdots r-l]s[i+1⋯r]=s[i+1−l⋯r−l]。

现在我们成功的把目标字符串的前缀转化到了一个我们已经求出 zzz 的位置。考虑这个时候直接调用 z[i+1−l]z[i+1-l]z[i+1−l] 就能得到此时的 zzz 函数值。

这时要分两种情况。如果这时的 zzz 值小于我们的字符串长度，那么当前的 zzz 值就是原来的 zzz 值。

而如果大于等于，我们就只能得到 z[i+1]≥z[i+1−l]z[i+1]\geq z[i+1-l]z[i+1]≥z[i+1−l] 的信息。那么就把 z[i+1]z[i+1]z[i+1] 暴力赋值成 r−ir-ir−i 后继续跑暴力。

复杂度不会证明，应该是 O(n)O(n)O(n) 的。

而如果要求其他串每个长度的后缀与原串的最长公共前缀，就要使用原串的 zzz 函数值来计算新串的信息，过程与上面相似。

border 理论与均摊分析

一些定理及其证明

首先我们来定义一些名词：

border：若 kkk 满足 s[1⋯k]=s[n−k+1⋯n]s[1\cdots k]=s[n-k+1\cdots n]s[1⋯k]=s[n−k+1⋯n]，则称 s[1⋯k]s[1\cdots k]s[1⋯k] 为 sss 的 border。

period：若 kkk 满足 s[1⋯n−k]=s[k+1⋯n]s[1\cdots n-k]=s[k+1\cdots n]s[1⋯n−k]=s[k+1⋯n]，那么 kkk 是 sss 的一个周期。

不难发现，如果 sss 有一个长为 kkk 的 border，那么就会有一个长为 n−kn-kn−k 的 period。

WPL

WPL，即弱周期引理。

它的内容是：若 p,qp,qp,q 均为 sss 的 period，且 p+q≤∣s∣p+q\leq|s|p+q≤∣s∣，则 gcd⁡(p,q)\gcd(p,q)gcd(p,q) 为 sss 的周期。

证明

由裴蜀定理，我们知道 ap+bq=gcd⁡(p,q)ap+bq=\gcd(p,q)ap+bq=gcd(p,q) 的解一定存在。

考虑模拟裴蜀定理证明的过程，钦定 p≤qp\leq qp≤q，那么有边界条件 p=1p=1p=1，此时该引理必然成立。

对于 1<p≤q1<p\leq q1<p≤q 的情况，假设我们已经完成了 (x,y)(x,y)(x,y) 的证明，其中 x<p\or y<q。由于 p+q≤∣s∣p+q\leq|s|p+q≤∣s∣，那么一定有 k∈[1,∣s∣]k\in[1,|s|]k∈[1,∣s∣]，k≤∣s∣−qk\leq |s|-qk≤∣s∣−q 和 k>pk>pk>p 其中一者已经被证明，手推一下不难发现上面两种情况可以推出 q−pq-pq−p 是 sss 的周期，通过辗转相除可得 gcd⁡(p,q)\gcd(p,q)gcd(p,q) 为 sss 的周期。

***P2336

这题实在太好了，看起来像一个很不起眼的字符串匹配裸题，但是却包含了非常多的思想。

首先我们发现建 AC 自动机的复杂度是 O(n∣Σ∣)O(n|\Sigma|)O(n∣Σ∣)，已经爆了，但是我们先不管如何优化，考虑如何统计答案。

首先来翻译问题：第一问是求一个模式串能由多少个文本串匹配，第二问是求一个文本串能匹配多少个模式串。我们将模式串的 AC 自动机建出来。

考虑正常暴力做法，第二问可以直接 O(∣T∣+∑∣S∣)O(|T|+\sum|S|)O(∣T∣+∑∣S∣) 匹配，然后贡献回退时间复杂度为 O(∑∣S∣)O(\sum|S|)O(∑∣S∣)，总复杂度爆炸。

既然暴力不行，那么我们按顺序来，考虑第一问怎么做。我们显然发现每个后缀可能会有不同串匹配上，那么如果下传 fail 树上标记的 id 或暴力跳 fail 统计的最坏时间复杂度均为 O(n2)O(n^2)O(n2)，无法通过。考虑优化。我们沿用 阿狸的打字机 一题的思路，对于我们走到的每一个前缀进行 fail 树上的操作，但是由于此题需要求贡献并而不是贡献和，所以需要一些 Trick 来除去多余贡献。

不难发现第一问就是 fail 树上到根链修改单点查询，转化为单点修改子树查，可以用树状数组维护。具体就是对于每个文本串暴力在 AC 自动机上跳，每个串跳到的所有点进行到根路径 +1 的操作，并将这一个串的所有贡献做并。然后最后询问每个模式串结尾节点得到答案。

这里使用一个 Trick：先对 AC 自动机上访问到的节点按 fail 树的 dfs 序排序，然后减去贡献就是相邻两个点的 LCALCALCA 到根的链上所有点 -1。很好证明，考虑原本不加贡献直接离线下来用 dfs 序排序，然后每次令这次询问的点与上次询问的点的 LCALCALCA 到这次询问的点的路径 +1，手玩一下发现这种做法是对的，然后其实这种做法和上面那种做法是一样的。

那么对于第一问，我们的思路就清晰了，对每个串先遍历一遍得到其 dfs 序，排序过后树状数组维护单点修改子树查询，最后对于每一个模板串进行查询贡献和。

模仿第一问，我们第二问的思路同样有了：同样是先遍历一遍得到其 dfs 序，然后在模式串的结尾先单点修改，然后对于每一个前缀进行到根链查询，可以转化为子树修改单点查。然后每次暴力进行每个前缀的查询，同样使用上面那个 Trick 减去多余的贡献，回退的话需要使用原来的 dfs 序全部做反操作。

然后最大的问题来了：建 AC 自动机的时间复杂度都爆炸了，你说怎么办呢？

可以使用一个小 Trick 来把建 AC 自动机的时间复杂度降为 O(nlog⁡∣Σ∣)O(n\log|\Sigma|)O(nlog∣Σ∣)，空间复杂度变为 O(nlog⁡∣Σ∣)O(n\log |\Sigma|)O(nlog∣Σ∣)。每次访问一个节点的儿子的复杂度变为 O(log⁡∣Σ∣)O(\log|\Sigma|)O(log∣Σ∣)。

考虑对每一个点都开一个动态开点线段树来存储 这个点的 x 指针在 Trie 上指向哪里，在 节点复制 那一步中利用可持久化的思想将两棵线段树合并在一起，具体就是同时遍历两棵线段树，如果此时该子树不存在，那么就直接将这个链接指向该子树，如果到了叶子，那么就可以直接把 fail 指向之前的那棵子树，查询时正常单点查询自己的后继即可。

子序列自动机

用来处理子序列问题，不太常用。

思想

考虑维护一个 nxti,cnxt_{i,c}nxti,c​ 指在第 iii 个位置之后字符 ccc 第一次出现的位置。这样可以让此时转移到 iii 状态时能够选择到下一个最近的位置，不难发现这样一定最优。

构建方式

字符集较小

当字符集较小时，我们可以倒序记录目前该位置及其以后每个字符集中的元素距离当前位置的最近的一个的位置，倒序更新每个点的 nxtnxtnxt 数组并更新字符集内部情况即可。

时空复杂度均为 O(n∣Σ∣)O(n|\Sigma|)O(n∣Σ∣)。询问复杂度 O(∣T∣)O(|T|)O(∣T∣)。

字符集较大

由于上面那种构建方式的时空复杂度均过大，考虑换一种思路。

我们用 vector 给每个字符集里的元素的出现位置开一个桶，初始将它排序，每次要查询时在某个桶里二分查找下一个位置即可。

构建复杂度为 O(∣Σ∣log⁡n)O(|\Sigma|\log n)O(∣Σ∣logn)，空间复杂度 O(n)O(n)O(n)，询问复杂度 O(∣T∣log⁡n)O(|T|\log n)O(∣T∣logn)。

板子题

直接套 字符集较大 时的做法即可。
